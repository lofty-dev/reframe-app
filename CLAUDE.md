\# 基本ルール



\- 常に日本語で応答すること。実装報告・調査結果・質問・エラーの説明も

&#x20; すべて日本語で書く。コード内のコメントも日本語でよい。

\- 例外：コミットメッセージは既存の形式に合わせる。


\## デプロイ後スモークテスト（必須）



mainへのpush（＝本番デプロイ）が発生したら、開発者が本番PWA上で

以下の4動線を必ず実施する。所要5分。機能追加・修正の内容に

かかわらず毎回同じ4つを回す。



1\. チェックイン：新規記録 → ホームに反映されること

2\. ストレス記録：新規記録 → 一覧に表示されること

3\. 伝えたいことメモ：作成 → チェック → 完了 → 診察等の記録に反映されること

4\. Bridge Session：開始 → セッションメモ入力 → メモに紐付け → 終了 →

&#x20;  診察等の記録の「言われたこと」に内容が保存されていること



この4つはStrideの生命線であり、特に4は2026-07-06に本番で

reply消失バグ（露出30分）を出した実績のある動線。



\### Claude Codeへの指示



\- 上記4動線に関わるコード（チェックイン保存、ストレス記録保存、

&#x20; toggleTellCheck / completeTellMemo / updateTellReply、

&#x20; Bridge Session関連）に変更を加えた場合は、実装報告の最後に

&#x20; 「⚠ スモークテスト対象の変更あり：動線◯番」と明記すること

\- 同一stateへの連続更新は必ず関数型 setState(prev => ...) を使う。

&#x20; レンダー時点のstate変数から新しい値を組み立てて set しない

&#x20; （2026-07-06のreply消失バグの原因パターン）
