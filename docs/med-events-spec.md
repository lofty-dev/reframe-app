# 実装指示書：「おくすりメモ」機能（服薬イベント記録）

Stride に服薬イベントを記録する機能を追加する。処方の変更や頓服の使用を「時系列上の点」として記録し、気分・睡眠グラフの注釈として表示する機能。

## 背景と設計思想（実装判断に迷ったらここに立ち返る）

- この機能は**服薬管理ではない**。毎日の服薬チェック・リマインダー・飲み忘れ集計・残薬管理は将来も含めて一切作らない。UI文言でも「管理」という語を使わない。機能名は「おくすりメモ」。
- 服薬イベントの価値は「気分・睡眠のグラフを読むときの注釈」にある。処方が変わった日とグラフの変化を、診察の場で突き合わせられるようにする。
- **効果の示唆を絶対にしない**。「増量後、気分が上昇しています」のような表示・集計・強調は禁止。グラフに印を打つだけで、読むのは診察室のユーザーと医師。
- 薬名マスタ・薬剤データベースは持たない。ユーザーが自分の呼び名（「頓服」「朝の薬」など）を自由入力する。見せる相手は処方した本人なので正式名称は不要。

---

## 1. データモデル

localStorage に新キー `stride_med_events` を追加。値はイベントオブジェクトの配列。

```json
{
  "id": "med_<timestamp>_<random>",
  "date": "2026-07-03",
  "type": "tonpuku",
  "label": "頓服",
  "note": "",
  "createdAt": "2026-07-03T14:00:00.000Z"
}
```

- `type` は `"start" | "increase" | "decrease" | "change" | "stop" | "tonpuku"` の6値。表示名は「開始／増量／減量／変更／中止／頓服使用」。定義は constants.js に置く。
- `date` はイベントが起きた日（YYYY-MM-DD）。過去日も選択可能（「先週変わったのを今記録する」ケースは普通にある）。未来日は不可。
- `label` は必須・最大30文字の自由入力。`note` は任意・最大100文字。
- 同日に複数イベントが存在してよい（一意制約なし）。

あわせて `stride_med_settings` を追加：

```json
{
  "quickTonpukuEnabled": false,
  "quickTonpukuLabel": "頓服"
}
```

storage.js に `loadMedEvents/saveMedEvents/loadMedSettings/saveMedSettings` と操作関数を、テーマ機能（stride_themes）で確立した命名・エラーハンドリング方針に合わせて実装する。文字数上限はテーマ同様データ層でもガードする。

### 既存機能への影響

- JSONエクスポート／インポート対象（ALL_STORAGE_KEYS）に `stride_med_events` と `stride_med_settings` を追加。旧バックアップにキーがなくてもエラーにしない。
- 服薬イベントは支援者に紐付かない（ユーザー本人の時系列データ）。支援者削除の影響なし。
- マイグレーション不要。

## 2. 入力導線

### 2-1. 医療タブ「おくすりメモ」（メイン導線）

医療タブに「おくすすりメモ」ではなく「おくすりメモ」の項目を追加する。

- 一覧画面：イベントを新しい順に時系列表示。各行は「日付・種別バッジ・ラベル・（あれば）メモ」。
- 追加：日付（デフォルト今日）→ 種別（6択）→ ラベル（自由入力、直近使ったラベルを候補チップとして最大5件表示し、タップで入力）→ メモ（任意）。
- 編集・削除可。削除は確認ダイアログを挟む。
- 空状態の文言：「処方が変わった日や頓服を使った日を記録しておくと、気分グラフと一緒に振り返れます」。

### 2-2. 頓服ワンタップ（ホーム画面）

- `quickTonpukuEnabled: true` のとき、ホーム画面のチェックインカード近くに小さなボタン「頓服を使った」を表示する。
- タップで即記録：`type: "tonpuku"`、`label: quickTonpukuLabel`、`date: 今日`。確認ダイアログは出さない。
- 記録直後にトースト「頓服の使用を記録しました［取り消す］」を数秒表示。誤タップの救済は確認ではなく取り消しで行う（ワンタップの価値を守るため）。
- 有効化の導線：おくすりメモで種別「頓服使用」のイベントを初めて保存したとき、「ホームにワンタップ記録ボタンを置きますか？」と1回だけ提案する（拒否したら二度と自動提案しない）。設定画面またはおくすりメモ画面からいつでもON/OFF可能。デフォルトOFF。
- ホームのボタンは装飾を抑え、チェックインより目立たせない。頓服を使わないユーザーには存在しない機能であること。

## 3. 表示：グラフへのアノテーション

グラフは2系統の自前実装が混在している。**統一・共通化のリファクタリングはしない**。それぞれの実装に合わせて最小の追記をする。

### 共通ルール

- グラフ内に描くのは**印のみ**。ラベル文字列をグラフ内に描画しない（14日/30日表示での文字衝突を避ける）。
- イベントの内容は**グラフ直下の凡例リスト**に出す：「● 6/12 増量・朝の薬」の形式で、表示期間内のイベントを古い順に列挙。期間内にイベントがなければ凡例ごと非表示。
- ホバー・ツールチップ・タップ詳細は実装しない（自前実装へのヒットテスト追加は複雑さに見合わない。凡例で足りる）。
- 効果を示唆する表示（前後の平均値比較・矢印・色分け強調など）を追加しないこと。

### 3-1. 記録詳細の気分グラフ(自前SVG折れ線、App.jsx:3702周辺)

- イベントがある日のx座標に**縦の点線**（`<line>` に stroke-dasharray）を描画。データ系列より背面、グリッド線より前面。
- 縦線の上端に小さな目印（`<circle>` r=3程度）。
- 同日に複数イベントがある場合も縦線は1本。凡例側で全件列挙する。
- 色はテーマのアクセント系から1色（ライト/ダーク両対応）。データ系列（気分の線）と明確に区別できること。
- 14日/30日どちらの表示期間でも動作すること。

### 3-2. Bridge Session内の気分・睡眠バー（divバーグラフ、App.jsx:1571周辺）

- イベントがある日の棒の**下（日付ラベル付近）に小さなドット**を表示する。棒の高さ計算には一切触れない。
- 凡例リストは気分・睡眠バーの下に1つだけ共通で表示（両方のバーに同じ日のドットが付く）。
- **Bridge Session表示設定に「おくすりメモ」トグルを追加。デフォルトOFF。**（服薬情報は機微度が一段高い。訪問看護師には見せるがデイケアスタッフには見せない等の使い分けを想定し、見せる相手ごとにユーザーが明示的にONにする）
- トグルOFF時はドットも凡例も一切描画しない。

### 3-3. 初版でやらないこと

- PDF出力への服薬イベント掲載（将来対応。既存PDFには手を触れない）
- おくすりメモ専用のグラフ・統計画面
- 頓服使用回数の集計表示（凡例の列挙のみ。「今月◯回」のような集計は効果示唆に近づくため作らない）

## 4. 発火・表示の交差点（実装前に必ず読むこと）

テーマ機能で「複数の導線が同じ状態に触る」見落としが2件出たため、本機能の交差点を先に列挙する。

| 交差点 | 仕様 |
|---|---|
| Bridge Session中にホームへ離脱し頓服ワンタップを押す | 通常どおり即記録＋トースト。ダイアログ等でセッションを妨げない。セッションに復帰したら（トグルONなら）ドット・凡例に即反映されてよい |
| 頓服ワンタップとチェックインの関係 | 完全に独立。頓服記録がチェックインの状態・導線に影響しない。逆も同様 |
| おくすりメモのイベント追加・編集・削除とグラフ | グラフ・凡例は表示時にstride_med_eventsを読むだけの一方向。イベント側からグラフへの通知・同期処理は不要 |
| テーマ機能との関係 | 交差なし。ThemePromptの発火条件に服薬イベントは一切関与しない |
| 過去日イベントの追加 | グラフの表示期間内なら次回表示時に反映される。それ以外の特別処理なし |
| 同日複数イベント | 一覧では全件表示、グラフでは印1つ＋凡例に全件 |

---

## 5. 文言・トーン規定

- 機能名は「おくすりメモ」で一貫。「服薬管理」「アドヒアランス」等は使わない。
- 頓服の記録に対して評価的なフィードバック（「今週は多いですね」等）を一切出さない。記録の確認トーストは事実の通知のみ。
- リマインダー・通知は実装しない。

---

## 6. 実装上の注意

- 既存のview命名・styles.jsの流用方針・constants.jsへの定数配置は、テーマ機能実装時の方針に合わせる。
- グラフ改修は既存の描画ロジック（座標計算・棒の高さ計算）に手を入れず、レイヤーの追加のみで実現すること。既存グラフの見た目がイベント0件のとき完全に無変化であることを必ず確認する。
- ダーク／ライト両テーマで、縦点線・ドット・凡例の視認性を確認する。

---

## 7. 受け入れチェックリスト

- [ ] おくすりメモから6種別のイベントを追加・編集・削除できる
- [ ] ラベル候補チップ（直近使用分）が出て、タップで入力される
- [ ] 未来日のイベントが作成できない
- [ ] 頓服イベント初回保存時にワンタップボタンの提案が1回だけ出る
- [ ] ワンタップ→即記録→トーストの取り消しで記録が消える
- [ ] ワンタップボタンは設定OFFで完全に消える
- [ ] 記録詳細の折れ線グラフに縦点線＋上端マーカーが出る（14日/30日両方）
- [ ] イベント0件のとき、既存グラフの見た目が完全に無変化
- [ ] グラフ直下の凡例に期間内イベントが古い順で列挙される
- [ ] Bridge Sessionのトグル（デフォルトOFF）でドット・凡例の表示が切り替わる
- [ ] Bridge Session中に離脱して頓服ワンタップ→復帰でセッション状態が保持され、記録も反映されている
- [ ] エクスポートに2キーが含まれ、インポートで復元される。キーのない旧バックアップでもエラーにならない
- [ ] ラベル30文字超・メモ100文字超が保存できない
- [ ] ダーク／ライト両テーマで表示崩れ・視認性の問題がない
- [ ] 「今月◯回」等の集計表示がどこにも存在しない（仕様どおりであることの確認）
